## 任务背景

Task 18–21 完成了 UI 美化的主体工作（DesignSystem、面板重构、设置页填充、OnboardingFlow 重设计），但遗留三项收尾工作：

1. `PanelStyle.swift`（旧面板常量）所有 token 已迁入 DesignSystem，但文件本身未删除
2. `MenuBarController` 仅有 Settings / Quit 两个菜单项，缺少外观子菜单和未配置提示功能
3. SwiftLint strict 模式下仍有 6 处违规

## 实施方案

### Step 1：查找并清理 PanelStyle/PanelColors 引用

通过 grep 确认两者只在自身文件中定义和引用，无其他源文件依赖，可直接删除。

### Step 2：删除 PanelStyle.swift

```bash
rm SliceAIKit/Sources/Windowing/PanelStyle.swift
```

### Step 3：重写 MenuBarController.swift

新功能：
- 继承 NSObject（满足 NSMenuDelegate 协议要求）
- 外观子菜单：三项（跟随系统/浅色/深色），通过 tag(0/1/2) 区分，调用 `ThemeManager.setMode(_:)`
- 实现 `NSMenuDelegate.menuWillOpen`：菜单弹出时同步勾选状态，确保与 ThemeManager 一致
- `refreshConfigStateIndicator()`：异步读取 configStore，判断 providers 是否为空
  - 为空：叠加紫色（#7C5CBF 近似）6pt 圆点在图标右上角（通过 NSImage.lockFocus 合成）
  - 非空：还原为基础图标

### Step 4：AppDelegate 集成

- `applicationDidFinishLaunching` 中创建 menuBarController 后立即调用 `refreshConfigStateIndicator()`
- `finishOnboarding` 中完成后再调一次，确保用户在 Onboarding 填写 Provider 后红点消失

### Step 5：修复 SwiftLint 违规

## 注意事项

- **手动验收待办**（无法命令行自动化）：
  - 划词后浮条弹出动画是否流畅
  - 小红点图标在 Retina 显示器上是否清晰（NSImage.lockFocus 方案是 1x 基础实现，Retina 下可能模糊；后续可改用 NSImage(size:flipped:drawingHandler:) 传入 isFlipped=false 并在 handler 里用 NSGraphicsContext 绘制）
  - Settings 里修改外观后，菜单栏子菜单勾选是否立即同步
  - Settings 里添加 Provider 后，红点是否消失（需手动触发 `refreshConfigStateIndicator`，当前仅在启动和 Onboarding 完成时触发；Settings 关闭时未触发）

- **潜在改进**：Settings 关闭窗口时可以触发一次 `refreshConfigStateIndicator`，可通过 NSWindowDelegate.windowWillClose 实现，但属于 Task 22 范围外的增量需求，留作后续。
- **红点图标质量**：当前使用 `lockFocus` / `unlockFocus` 方案，在高 DPI 屏幕上可能输出 1x 位图。如果实际验收发现模糊，改用 `NSImage(size:flipped:drawingHandler:)` 方案。
